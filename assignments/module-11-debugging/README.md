# Module 11 – Debugging and Defensive Programming in R

## Overview

Worked on identifying, reproducing, and fixing errors in R code, along with adding defensive programming checks to improve reliability.

## Key Concepts

- Reproducing errors using test data
- Understanding logical operators (`&&` vs `&`)
- Debugging logical vs element-wise operations
- Writing safer code with input validation (`stop()`)
- Using loops with `seq_len()` for safer iteration
